# Generic Preset · 通用兜底

## 1. 适用场景

主题不明确属于竞品 / 行业 / 人物等任何专门类型时使用。**比 competitor preset 更弹性，但章节精度更低**——如果能判明类型，优先用专门 preset。

典型场景：
- 一个含混的概念（"web3 钱包安全现状"）
- 一个跨界主题（"医疗 AI + 患者教育"）
- 一段时间窗的快照（"2026 年 4 月各家大模型对比"）
- 一个没明确标的的方向（"Agent 框架最近半年的进展"）

---

## 2. 子 Agent prompts（3 个并行，按需可降到 2 个）

### 输出规范（所有子 Agent 必须遵循 · inline 到每个 prompt 末尾）

> 读者是外行，常调研自己不懂的领域。所以：
> 1. **专业术语首次出现加括号白话注解**——金融词 + 技术词 + 领域黑话都算（凡外行不秒懂的）。例：ARR（一年能收多少订阅费）、RAG（让 AI 先查资料再回答，类似开卷考试）
> 2. **复杂的机制 / 原理必须打比方**，配一句"就像……"。例：飞轮效应（像滚雪球，越转越大）
> 3. **非美元数字加 USD 锚点**（原币 + 括号约合美元）。引用别人原话保留原文不改
> 4. **完整结果落盘**：把完整调研结果（含全部来源链接）写入主 Agent 指定的文件（如 `~/.cache/ray-deep-research/{slug}/agent-a.md`），返回消息只写 ≤500 字核心摘要 + 文件路径（主 Agent dispatch 时替换为实际路径）

### 子 Agent A · 主流玩家 / 头部信源扫描

```
你是一名研究员。任务是：扫描 {{TOPIC}} 这个主题下的主流玩家、头部信源、核心事实。

{{BROWSER_ACCESS}}

需要拿到：
1. 该主题下被公认为权威/重要的 5-10 个玩家、产品、人物或概念
2. 每个的核心信息（一句话定位 + 关键数据 + 主要争议点）
3. 该主题最近 6-12 个月的关键事件 / 转折点
4. 该主题的核心数据（市场规模、用户数、增速、投资额等可量化指标）

来源优先级：
- 权威媒体（TechCrunch / The Verge / Bloomberg / 36氪 / 虎嗅）
- 一手数据（官网 / 公司公告 / 招股书）
- 行业报告（Gartner / a16z 趋势文 / 头部分析师博客）

输出要求（中文，结构化）：
- 主流玩家列表（含一句话评价 + 来源链接）
- 关键事件时间线（最近 6-12 个月）
- 核心数据
- 认知盲区（哪些重要问题信源没回答）
```

### 子 Agent B · 真实用户 / 一线声音

```
你是一名研究员。任务是：在 {{TOPIC}} 这个主题下，找到真实用户/从业者/批评者的一线声音。

{{BROWSER_ACCESS}}

需要覆盖（按主题相关性挑选）：
1. **Reddit** — 找相关 sub 看用户实际反馈
2. **Twitter / X** — 找 KOL 和真实用户的近期发言
3. **HackerNews** — 看技术圈讨论
4. **知乎 / 少数派 / 即刻** — 中文圈观点
5. **B站 / YouTube** — 评测视频或 vlog
6. **小红书** — 大众/生活化视角（如主题与日常相关）

输出要求（中文，结构化）：
- 各平台典型声音（带来源链接）
- 用户最关心的问题（Top 5）
- 用户最常踩的坑 / 抱怨
- 与官方叙事的主要差异
- 中外认知差异（如有）
```

### 子 Agent C · 趋势 / 反思 / 批评视角

```
你是一名研究员。任务是：从更高维度看 {{TOPIC}} — 趋势在哪、反对意见是什么、潜在风险是什么。

{{BROWSER_ACCESS}}

需要拿到：
1. 行业分析师 / 投资人 / 领先思考者对该主题的判断（看长文 / 播客 / 访谈）
2. 反对意见和质疑（看批评者文章 / Reddit 反对帖 / Twitter 争议线）
3. 与历史类似主题的对比（这次和上次有什么不同？）
4. 12-24 个月可能的演化方向
5. 隐藏的风险点（监管 / 技术 / 商业模式 / 信任）

输出要求（中文，结构化）：
- 多空双方主要论点
- 与历史的对比 / 教训
- 未来 12-24 个月可能演化（3 种场景）
- 隐藏风险点
- 我（如果作为这个主题的参与者）该警惕什么
```

## 3. 章节结构（HTML 报告 7-8 章，弹性）

按主题特性挑章节，但保持顺序：

1. **Hero** — 一句话核心结论
2. **Quick Facts 速览板** — 8-12 格，按主题挑标签
3. **TL;DR · 一页纸结论** — 三段：现状 / 关键张力 / 核心建议
4. **主流玩家 / 关键事实** — 列表 + 关键事件时间线
5. **真实声音** — 多平台用户/从业者评价 + 用户原话引用
6. **趋势与反思** — 多空论点对比 + 历史教训
7. **风险与盲点** — 多 callout 块呈现
8. **决策建议** — 按主题给出可操作建议（不一定是 Path A/B/C 三选一，可以是 5-7 条 actionable insights）

---

## 4. Quick Facts 字段（按主题挑 8-12 格）

候选标签：
- 主题成熟度（萌芽 / 成长 / 成熟 / 衰退）
- 核心玩家数量
- 市场规模 / 增速
- 关键人物（如有）
- 最近重大事件
- 主要争议点
- 中外认知差距
- 时间窗口（3 个月 / 12 个月 / 3 年）
- 风险等级
- 我的判断（红色 verdict cell）

---

## 5. 决策建议格式（最后一章）

弹性结构：

```
### 7 个 Actionable Insights（编号列表）
每个 insight：现象 → 解读 → 我能做什么

### 不论怎么参与，要避开的 3 个坑

### 最后一句（callout 块 + 引用样式）
对主题做一个升华的短判断
```

---

## 与 competitor preset 的关键差异

| 维度 | competitor | generic |
|------|-----------|---------|
| 子 Agent 数 | 4 | 3 |
| 章节数 | 12 | 7-8 |
| Quick Facts | 12 格固定 | 8-12 格按需 |
| 决策建议 | Path A/B/C 三选一 + 详细行动卡 | 7 个 insights + 避坑清单 |
| 重点 | 复刻可行性 | 可操作判断 |
