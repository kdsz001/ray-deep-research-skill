# Competitor Preset · 竞品调研

## 1. 适用场景

用户想做一个产品，想看看现有玩家是谁、活得怎么样、能不能复刻、从哪儿切入。**默认 preset，覆盖面最广，对错都有用**。

触发信号：
- "调研 X 这个产品"
- "想做一个跟 X 类似的"
- "X 和 Y 哪个好"
- "对比 X 的几个竞品"
- 给出一个产品官网 / App 链接

---

## 2. 子 Agent prompts（4 个并行）

**调度规则**：4 个子 Agent 同时 dispatch + run_in_background=true。每个子 Agent 必须包含"必须加载 web-access skill 并遵循指引"。每个 prompt 把"{{TARGET}}"替换为调研主题。

### 输出规范（所有 4 个子 Agent 必须遵循 · 必须 inline 到每个 prompt 末尾）

> **读者画像**：产品型创业者，不是职业投资人。所以输出必须：
>
> 1. **所有专业术语首次出现加括号白话注解**——不只金融商业词（种子轮、估值、ARR、PMF、CAC、LTV、Pre-IPO、ICP 等），**也包括技术词和领域黑话**（RAG、模型蒸馏、Agent、套壳、扩散模型等，凡外行不秒懂的都算）。例：种子轮（公司最早期融资）、ARR（一年能收多少订阅费）、套壳（买别人的 AI 模型套个壳，类似贴牌）
> 2. **复杂的机制 / 商业模式 / 技术原理必须打比方**。读完定义仍会懵的地方，配一句"就像……"。例：飞轮效应（就像滚雪球：用户越多→数据越多→产品越好→更多用户）
> 3. **所有财务数字必须有 USD 锚点**。原币 + 括号 USD 等价。例：月付 ¥84（约 $11.7）、估值 ¥2000 亿（约 $1.4B）、年薪 ¥1500 万（约 $10.4 万）。参考汇率：$1 ≈ ¥7.2（人民币）≈ ¥145（日元）≈ ₩1340（韩元）≈ €0.92 ≈ £0.78（按调研当时实际汇率，可估算）
> 4. **引用别人的原话保留原文不改**（用户评论、KOL 推文、媒体报道在引号内的内容），只对子 Agent 自己写的总结/分析部分应用规范
> 5. **完整结果落盘**：把完整调研结果（含全部来源链接）写入主 Agent 指定的文件（如 `/tmp/ray-research/{slug}/agent-a.md`），返回消息只写 ≤500 字核心摘要 + 文件路径

**主 Agent 在 dispatch 子 Agent 时，必须把上面这 5 条作为一段附在每个子 Agent 的 prompt 末尾**（第 5 条要替换成该 Agent 的实际文件路径）。这样所有调研结果回来就已经预处理好，主 Agent 整合时不需要再 reformat。

### 子 Agent A · 海外英文社区口碑

```
你是一名竞品调研员。任务是：全面收集 {{TARGET}} 在英文社区的真实用户评价和市场反馈。

必须加载 web-access skill 并遵循指引。

需要覆盖的平台：
1. **Product Hunt** — 搜该产品主页 + 评论页 + launches + awards 子页 — 拿到上榜情况、当日排名、获奖、所有评论的好评/差评要点、Maker 的回应。**重点核实"App of the Year"等奖项是否真实存在**（PH 只有 Launch Awards 这一类）
2. **Reddit** — 搜产品名 + 相关 sub（r/macapps / r/iosapps / 行业相关 sub），找到所有讨论帖，提取真实用户的使用体验和与竞品对比帖
3. **Hacker News** — 通过 hn.algolia.com 搜，看是否有 Show HN 或讨论
4. **Twitter / X** — 搜官号 + 创始人，看推文里透露的用户量、增长、融资、团队信息
5. **YouTube** — 找 top 评测视频，提取 KOL 测评要点；标注是否含 affiliate link
6. **App Store / Play Store** — 评分、评论数、最新更新、用户吐槽（用 itunes lookup API 拿一手数据）
7. **Trustpilot** — 域名 review，注意有时多个域名都要查（如 product.com 和 product.now）

输出要求（中文，结构化）：
- 每个平台的关键发现（带来源链接）
- 用户最爱的 3 件事
- 用户最骂的 3 件事
- 团队/融资/用户量的任何线索
- 与竞品对比中提到的优劣
- 总结：在英文世界的真实口碑评分（1-10）+ 一句话定性

避免编造。没找到就说没找到。优先一手来源。
不要用搜索引擎缓存的二手摘要——直接打开真实页面读评论。Reddit、PH 评论用 CDP 浏览器读。
```

### 子 Agent B · 中文社区口碑

```
你是一名竞品调研员。任务是：调研 {{TARGET}} 在中文世界的认知度、口碑和用户反馈。

必须加载 web-access skill 并遵循指引。

需要覆盖的平台（这些都需要 CDP 浏览器访问，反爬严重）：
1. **小红书** — 搜产品名 + 类目相关词，提取 KOL 和素人的使用体验
2. **知乎** — 搜产品名 + 类目相关词，找到回答和文章
3. **B站** — 搜评测视频
4. **即刻** — 看创业者/产品圈的讨论
5. **微信公众号 / 36氪 / 虎嗅 / 少数派** — 中文 KOL/媒体报道（少数派 sspai.com 重点查）

同时调研：
- 中国市场的同类竞品现状（大厂 + 创业 + 开源三阵营）
- 该产品是否在中国可用（要梯子吗？是否做了中文优化？中文识别效果如何？）
- 中文用户对此类产品的需求和痛点
- 中文用户付费意愿评估

输出要求（中文，结构化）：
- 每个平台的关键发现（带来源链接）
- 在中文圈的认知度（高/中/低/几乎零）
- 中文用户对它的评价
- 中文市场的同类竞品和市场格局
- 给中国创业者的启示：中国市场是机会还是雷区？

避免编造。没找到就说没找到——这本身就是重要信息。
```

### 子 Agent C · 竞品横向对比

```
你是一名竞品调研员。任务是：对 {{TARGET}} 所在赛道做完整横向对比。

必须加载 web-access skill 并遵循指引。

第一步：识别 5-8 个主要竞品（必须包含赛道明确的 Top 1-3）。来源：{{TARGET}} 官网的对比页 / G2 / Capterra / Reddit 上的"X vs Y"帖 / 行业评测文章。

每个竞品需要拿到：
- 官网定位 + Slogan
- 核心功能差异（vs {{TARGET}}）
- 平台支持
- 定价（免费档/付费档）
- 底层技术（用了哪个模型 / 是不是套壳 / 有没有自研）
- 团队规模 + 融资（最近 12 个月轮次）
- 用户量级 / 流量级别
- 主要差异化卖点
- 用户对它的主要好评和差评

也调研：
- 这个赛道最近 12 个月的整体动态（重大融资、新玩家、技术升级）
- 谁是真正的 Top 3
- {{TARGET}} 在这个格局里的位置（领先？追赶？差异化？）
- 留给后来者的机会窗口（垂直、本地化、隐私、深度集成等）

输出要求（中文，结构化）：
- 一张横向对比表格（Markdown）：产品 / 平台 / 定价 / 技术 / 融资 / 主要卖点 / 用户评价
- 每个竞品 200-300 字简介
- 整个赛道的市场格局和趋势分析
- {{TARGET}} 相对竞品的优势和劣势
- 给后来者的 5-8 个真实机会窗口

优先去官网看真实信息，搜索引擎只用来发现 URL。
```

### 子 Agent D · 团队 / 融资 / 流量 / 用户量

```
你是一名竞品调研员。任务是：挖出 {{TARGET}} 的公司、团队、融资、流量、用户量级信息。

必须加载 web-access skill 并遵循指引。

需要查的：
1. **公司/创始人**
   - 官网 about / footer / 隐私政策 / 服务条款 — 找公司名、地址、创始人
   - LinkedIn 搜公司名和创始人 — 团队规模、背景
   - Crunchbase / PitchBook / Tracxn — 融资记录
   - GitHub — 是否有开源代码 / 创始人 GitHub
   - 招聘信息能反映团队规模和方向

2. **流量 & 用户量级**
   - SimilarWeb 上该域名的月访问量、流量来源（直接/搜索/社交/推荐）、停留时长、跳出率、地理分布
   - App Store / Play Store 评论数 + 排名 倒推下载量级
   - Sensor Tower（sensortower.com，data.ai 已并入它）上是否有公开数据
   - Twitter / X 上创始人/官号的粉丝数和增长

3. **媒体报道 & 背书**
   - TechCrunch、The Verge、Wired、Bloomberg 等大媒体是否报道过
   - 任何顶级 VC 的公开背书
   - YC / Techstars / 其他加速器的校友身份

4. **增长信号**
   - 公开提到的用户数、收入数据（Twitter / 创始人访谈 / 招聘 JD）
   - 招聘 JD 反映的扩张速度

输出要求（中文，结构化，带原始来源链接）：
- 公司基本档案
- 团队画像（创始人背景、团队规模估算）
- 融资史（每一轮的金额、投资方、估值）
- 流量数据（月访问量、来源结构、地理分布）
- 用户量级估算（含估算逻辑）
- 关键背书和媒体报道
- 增长信号
- 一句话总结：这是一家什么阶段的公司

找不到的就说找不到。不要瞎编数字。
**特别注意**：搜索引擎里常出现 AI 摘要给出错误的融资数字（混淆同名公司或当前公司与已倒闭公司），所有数字必须从一手数据库（Crunchbase/PitchBook/Tracxn）或权威媒体公告确认。
```

## 3. 章节结构（HTML 报告 12 章，其中第 9 中国市场、第 10 隐私争议两章按需出现）

按此顺序：

1. **Hero** — 一句话核心结论 + 副标题（带情绪、带强调色）
2. **Quick Facts 速览板** — 12 格黑色 grid（10 秒看完，最后一格红色 verdict）
3. **TL;DR · 一页纸结论** — 一段事实概述 + 黑色 TL;DR 块（"如果你想做和它一样的产品，三种结局" + "如果你只能记住三件事"）
4. **公司档案** — 创始人、融资、团队、合规进度，含融资真伪提醒
5. **产品本身** — 定位、核心功能、平台、定价（stats 卡片）、底层技术
6. **市场反馈** — 多平台评分条 + Top 3 好评 + Top 3 差评（two-col 卡片）+ 用户原话引用
7. **用户群体** — 规模 stats + 地理分布 metric-list + 画像 + 获客路径
8. **竞品格局** — 赛道资本水位 + 横向对比表格（Top 3 标签 + 调研对象高亮行）
9. **中国市场**（如目标产品涉及）— 中文渠道认知度 metric-list + 中国用户态度引用 + 中国竞品全景 + 给中国创业者判断
10. **隐私争议**（如有）— 事件时间线 + 营销话术 vs 实际行为对比 + 对后来者的启示
11. **复刻可行性** — 各维度 score-card（技术 / 数据 / 资金 / 营销 / 时间窗口 / 综合）+ 8 个真实机会窗口
12. **决策建议** — 三条路径黑色 verdict 块 + Path A/B/C 详细行动卡片 + 通用避坑清单 + 最后一句

---

## 4. Quick Facts 字段（12 格）

| 位置 | 标签 | 数据形态 |
|------|------|---------|
| 1 | 阶段 | 文字（如"种子轮"） |
| 2 | 融资估算 | 美元区间或具体数 |
| 3 | 团队 | 人数区间 |
| 4 | 月访问量 | M / K |
| 5 | 移动下载量 | K+ / M+ |
| 6 | 付费投放占比 | % |
| 7 | Pro 定价 | $/月 |
| 8 | App Store 评分 | x.xx/5 |
| 9 | Trustpilot 评分 | x.x/5 |
| 10 | vs 头部对手资本比 | 1:N |
| 11 | 复刻可行性 | x/10 |
| 12 | **最终判断**（红色 verdict cell） | 文字（如"不建议正面复刻"） |

如某字段无数据 → 标"未披露"，不要留空。

---

## 5. 决策建议格式（最后一章）

固定结构：

```
### 三条决策路径（黑色 verdict 块 + 3 个 verdict-option 子卡片）
- Path A · 低风险：不做，做隔壁
- Path B · 中风险：做垂直版
- Path C · 高风险：正面挑战

### Path A · 30 天行动指南（绿色 border 卡片 ×2）
- 卡片 1：第 1-15 天（验证替代方向）
- 卡片 2：第 16-30 天（产品定义）

### Path B · 30 天行动指南（编号列表 ×5）

### Path C · 30 天行动指南（红色 border 卡片 ×2）
- 卡片 1：必备前置条件（4 项）
- 卡片 2：30 天关键决策

### 不论选哪条路，都要避开的 3 个坑（无序列表）

### 最后一句（callout 块 + 引用样式）
"你最大的优势不是资金，是你不需要赢。"——升华到创业者本身
```
