# Investment Preset · 投资决策调研

## 1. 适用场景

用户想判断一个**可投资标的**值不值得投：股票（美股/港股/A股）、IPO 打新、加密币（投资决策类，非即时操作）、基金/ETF、Pre-IPO 公司。**终点不是"能不能复刻"，而是"该不该买、买多少、什么情况下改判断"**。

触发信号：
- "X 值不值得买 / 该不该投 / 能不能上车"
- "分析 X 这只股票 / 帮我看看 X 的投资价值"
- "X 要 IPO 了，打不打新"
- "调研 X"且 X 是上市公司 / 拟上市公司 / 主流加密资产
- 给出 ticker（如 SPCX、2382.HK、$BTC）

**与其它 preset 的边界**：调研意图是"做一个类似产品"→ competitor；是"看懂这个行业"→ generic/industry；是"这钱投进去会怎样"→ 本 preset。加密项目的合约安全 / 土狗快筛 / 撸毛 → ray-crypto skill（不进本流程）。

**标的是加密资产时的字段映射**：财报 → 链上数据 + 代币经济学（供应释放 / 解锁表 / 协议收入）；SEC filings → 项目文档 + 审计报告；机构持仓 → 巨鲸地址 + ETF 持仓。其余结构不变。

## 2. 子 Agent prompts（4 个并行）

**调度规则**：4 个子 Agent 同时 dispatch + run_in_background=true。每个 prompt 把"{{TARGET}}"替换为标的名（含 ticker）、把"{{BROWSER_ACCESS}}"替换为 `environment.md` 按当前模式给出的浏览器能力指令段。

### 输出规范（所有 4 个子 Agent 必须遵循 · 必须 inline 到每个 prompt 末尾）

> **读者画像**：产品型创业者做个人投资决策，不是职业投资人。所以输出必须：
>
> 1. **所有专业术语首次出现加括号白话注解**——金融词（PE、EV/EBITDA、TTM、做市、13F、绿鞋机制等）+ 技术词 + 领域黑话，凡外行不秒懂的都算。例：PE（股价是每股年利润的多少倍，越高越贵）、13F（美国机构每季度必须公开的持仓清单）
> 2. **复杂的机制 / 商业模式 / 估值逻辑必须打比方**。读完定义仍会懵的地方，配一句"就像……"。例：绿鞋机制（就像演唱会主办方多备 15% 的票，太火就加卖、遇冷就回购托底）
> 3. **所有财务数字必须有 USD 锚点**。原币 + 括号 USD 等价。例：营收 ¥380 亿（约 $53 亿）、目标价 HK$80.67（约 $10.3）
> 4. **引用别人的原话保留原文不改**（分析师观点、社区评论、财报电话会原话），只对子 Agent 自己写的总结/分析部分应用规范
> 5. **完整结果落盘**：把完整调研结果（含全部来源链接）写入主 Agent 指定的文件（如 `~/.cache/ray-deep-research/{slug}/agent-a.md`），返回消息只写 ≤500 字核心摘要 + 文件路径
> 6. **时效性数据必须带"截至日期"**：股价 / 市值 / 估值倍数逐条标注数据日期——投资数据过期一周就可能误导决策

**主 Agent 在 dispatch 子 Agent 时，必须把上面 6 条作为一段附在每个子 Agent 的 prompt 末尾**（第 5 条替换成该 Agent 的实际文件路径）。

### 子 Agent A · 财报与基本面

```
你是一名基本面分析员。任务是：搞清 {{TARGET}} 的真实财务状况和赚钱能力。

{{BROWSER_ACCESS}}

需要拿到（优先一手来源）：
1. **财务三表核心数据**（最近 2-3 个财年 + 最近 4 个季度）：营收、毛利率、净利润、经营现金流、资产负债结构。来源优先级：公司 IR 页 / SEC filings（10-K / 10-Q / S-1 招股书）/ 港交所披露易 / 巨潮资讯 > 财经数据网站
2. **收入结构**：按业务分部 / 地区拆分，哪块赚钱哪块烧钱，增速分别多少
3. **盈利质量**：利润是真金白银还是纸面富贵？（经营现金流 vs 净利润、应收账款趋势、一次性损益占比、政府补贴依赖度）
4. **管理层指引**：最近财报电话会 / 招股书里对未来的承诺（Motley Fool / Seeking Alpha 有 transcripts）
5. **未上市或 IPO 标的**：招股书关键数据 + 融资与估值史 + 基石投资者名单

输出要求（中文，结构化，每个数字带来源和数据日期）：
- 财务健康画像（一句话定性 + 关键数字表）
- 收入结构与各业务增速
- 盈利质量评估（1-10 分 + 理由）
- 三个最值得警惕的财务信号
- 管理层最近的关键承诺（用于后续追踪兑现）

避免编造。找不到就说找不到。所有数字必须从一手文件（财报原文/招股书）或权威财经媒体确认，不要信搜索引擎 AI 摘要给的数字。
```

### 子 Agent B · 行业格局与竞争

```
你是一名行业分析员。任务是：搞清 {{TARGET}} 所在赛道的格局，以及它守不守得住自己的位置。

{{BROWSER_ACCESS}}

需要拿到：
1. **赛道基本盘**：市场规模、增速、当前处于周期什么位置（扩张/见顶/出清/复苏）
2. **竞争格局**：识别 3-6 个主要竞对，每个拿到市占率 / 营收量级 / 核心差异 / 最近 12 个月动作
3. **护城河检验**：{{TARGET}} 靠什么挡住竞争（技术壁垒 / 规模效应 / 网络效应 / 转换成本 / 牌照资质）？这些壁垒最近是在变宽还是变窄？
4. **上下游议价权**：对供应商和客户谁更强势？大客户集中度？
5. **颠覆风险**：有没有新技术 / 新模式 / 新玩家可能掀桌子？

输出要求（中文，结构化，带来源链接）：
- 一张竞争格局横向对比表（公司 / 市占率或量级 / 核心优势 / 最近动向）
- {{TARGET}} 在格局中的位置（领先者 / 追赶者 / 差异化生存者）
- 护城河评估（1-10 分 + 变宽还是变窄的判断依据）
- 行业周期判断 + 对 {{TARGET}} 未来 12-24 个月的含义

避免编造。行业数据优先引用具名机构报告（Gartner / IDC / 中金 / 招股书行业章节），标注出处。
```

### 子 Agent C · 市场情绪与机构观点

```
你是一名市场情绪分析员。任务是：搞清市场上的聪明钱和散户分别怎么看 {{TARGET}}。

{{BROWSER_ACCESS}}

需要拿到：
1. **分析师共识**：评级分布（X 买入 / Y 中性 / Z 卖出）+ 目标价区间 + 共识目标价。来源：TipRanks / Yahoo Finance / MarketBeat / 富途 / 雪球
2. **机构动向**：13F 持仓变化（美股）/ 港股通持股变化 / 大股东与内部人最近的增减持记录
3. **散户情绪**：Reddit（r/stocks、r/investing、标的相关 sub）、Seeking Alpha 评论区、X/Twitter、雪球、富途评论区——多头和空头各自的核心论点
4. **股价与事件对应**：最近 6-12 个月股价关键拐点分别对应什么事件
5. **IPO 标的**：认购热度、灰市价格、基石投资者质量、同类 IPO 近期首日表现

输出要求（中文，结构化，带来源链接和数据日期）：
- 机构评级分布 + 目标价 vs 现价的隐含涨跌幅
- 聪明钱最近一个季度是在进还是在退（含具体机构名）
- 多头 Top 3 论点 vs 空头 Top 3 论点（各自原话引用 1-2 条）
- 情绪定性：过热 / 乐观 / 分歧 / 冷清 / 恐慌
- 共识在哪里、分歧在哪里——分歧点往往就是决定输赢的点

避免编造。找不到就说找不到。评级和持仓数据必须标注截至日期。
```

### 子 Agent D · 风险与空头论点

```
你是一名专职唱反调的风险分析员。任务是：找出 {{TARGET}} 最强的反方证据——如果这笔投资会亏钱，最可能因为什么。

{{BROWSER_ACCESS}}立场设定：你的 KPI 是找到真实的、有杀伤力的风险，不是平衡报道。

需要拿到：
1. **做空报告与负面研究**：浑水 / Hindenburg / Grizzly 等是否发过报告？核心指控是什么？公司如何回应？后续验证如何？
2. **监管与诉讼**：正在进行的诉讼、监管调查、合规风险（含中概股/地缘政治因素，如适用）
3. **财务疑点**：审计意见异常、频繁更换审计师、关联交易、商誉占比、存货异动、大股东质押
4. **估值泡沫检验**：当前估值倍数 vs 同行 vs 自身历史区间，处于什么分位？隐含了多高的增长预期？这个预期现实吗？
5. **业务脆弱点**：客户集中、单一产品依赖、关键人物风险、技术路线被颠覆的可能
6. **加密标的额外查**：代币解锁表（未来 12 个月抛压）、团队钱包动向、协议收入真实性

输出要求（中文，结构化，带来源链接）：
- 最强空头论点 Top 5，每条含：论点 + 证据 + 杀伤力评估（致命 / 重伤 / 皮外伤）+ 多头的反驳（如有）
- 最坏情景推演：什么组合发生时，跌 30% / 腰斩 / 归零
- 哪些风险已经被股价消化、哪些还没有
- 一句话总结：这笔投资最大的一个雷是什么

避免编造，但也不要因为"找不到实锤"就放过合理怀疑——标注"疑点，未证实"即可。
```

## 3. 章节结构（HTML 报告 12 章，其中第 11 中国投资者实操按需出现）

按此顺序：

1. **Hero** — 双问句式核心结论：**"是不是好生意？是/否。现在是不是好价格？是/否。"**（好公司 ≠ 好股票，这个二分是投资报告的灵魂）
2. **Quick Facts 速览板** — 12 格黑色 grid（见下节），最后一格红色 verdict
3. **TL;DR · 一页纸结论** — 三句话概括 + 黑色 TL;DR 块（"如果你只能记住三件事"）
4. **公司与业务** — 商业模式（打比方讲清怎么赚钱）、收入结构、治理与管理层
5. **财务表现** — 营收/利润/现金流趋势（metric-list 或时间线）+ 盈利质量评估 + 财务警惕信号
6. **行业与竞争格局** — 赛道周期位置 + 竞对横向对比表（调研对象高亮行）+ 护城河评估 score 条
7. **增长驱动与催化剂** — 未来 12-24 个月的关键事件时间线（财报日 / 产品发布 / 政策节点 / 解锁日）
8. **估值** — 横向（vs 同行）+ 纵向（vs 自身历史）对比表 + 当前价格隐含了什么预期 + 敏感性（乐观/中性/悲观三档对应股价）
9. **机构观点与市场情绪** — 评级分布 stats + 聪明钱动向 + 多空论点 two-col 卡片 + 原话引用
10. **风险与空头论点** — Top 5 空头论点（含杀伤力标签）+ 最坏情景推演 callout
11. **中国投资者实操**（如标的可被中国个人投资者交易）— 购买通道（港美股券商 / 港股通 / 合规性）+ 汇率与税务注意 + 中文社区对它的认知偏差
12. **投资决策** — 三条路径黑色 verdict 块 + 触发条件清单 + 通用避坑 + 免责声明 + 最后一句

## 4. Quick Facts 字段（12 格）

| 位置 | 标签 | 数据形态 |
|------|------|---------|
| 1 | 标的 | ticker + 市场（如 NASDAQ: SPCX） |
| 2 | 市值 / 估值 | USD（标注截至日期） |
| 3 | 现价 / 发行价 | USD + 52 周位置 |
| 4 | 营收（TTM） | USD + 同比增速 |
| 5 | 盈利状态 | 净利率 % 或 "亏损（烧钱速度）" |
| 6 | 估值倍数 | PE / PS / EV-EBITDA 按标的挑最合适的一个，附同行中位数 |
| 7 | 现金与负债 | 现金储备 + 净负债率 |
| 8 | 机构共识 | X 买 / Y 中性 / Z 卖 + 目标价 |
| 9 | 聪明钱动向 | 加仓 / 减仓 / 分歧 |
| 10 | 最近催化剂 | 一句话 + 日期 |
| 11 | 综合评分 | x/10 |
| 12 | **最终判断**（红色 verdict cell） | 文字（如"好公司，等回调"） |

如某字段无数据 → 标"未披露"，不要留空。加密标的按第 1 节的字段映射替换（如营收 → 协议收入、机构共识 → ETF/巨鲸动向）。

## 5. 决策建议格式（最后一章）

固定结构：

```
### 三条参与路径（黑色 verdict 块 + 3 个 verdict-option 子卡片）
- Path A · 保守：不参与 / 回避，把钱放在哪里更好
- Path B · 稳健：观望 + 触发条件成立再进（或小仓位定投）
- Path C · 进取：现在建仓，配多少仓位、怎么分批、止损位在哪

### 触发条件清单（callout 块 · 决策的"活"的部分）
- 升级判断的信号：出现什么（如毛利率连续两季回升 / 空头指控被证伪）→ 从观望转买入
- 降级判断的信号：出现什么（如大股东减持 / 指引下修）→ 从持有转卖出
- 每条触发条件必须可观察、可验证（财报日 / 公告 / 数据源明确）

### 不论选哪条路，都要避开的 3 个坑（无序列表）

### 免责声明（mono 小字）
本报告为个人研究用途的信息整理，不构成投资建议。数据截至 {调研日期}，投资有风险，决策需独立判断。

### 最后一句（callout 块 + 引用样式）
升华判断，例："你赚不到认知以外的钱，但可以先把认知补上。"
```

## 与 competitor preset 的关键差异

| 维度 | competitor | investment |
|------|-----------|------------|
| 核心问题 | 我能不能做一个 | 我该不该买 |
| Hero 框架 | 一句话结论 | 好生意 × 好价格 双问 |
| 子 Agent D | 团队/融资/流量 | 专职唱反调（空头论点） |
| 决策章 | Path A/B/C（做不做） | 三档参与路径 + 触发条件清单 |
| 特有章节 | 复刻可行性 | 估值 + 情景推演 + 中国投资者实操 |
| 特有纪律 | — | 时效数据带截至日期 + 免责声明 |
